# Internationalization (i18n)

The platform supports **6 languages** with client-side switching, no URL changes, and automatic RTL for Arabic.

---

## Supported Languages

| Code    | Language           | Direction     |
| ------- | ------------------ | ------------- |
| `en`    | English            | LTR (default) |
| `ar-EG` | Arabic (Egypt)     | RTL           |
| `de`    | German (Deutsch)   | LTR           |
| `es`    | Spanish (Español)  | LTR           |
| `it`    | Italian (Italiano) | LTR           |
| `fr`    | French (Français)  | LTR           |

---

## Architecture

All i18n code lives in `shared/` so both `public-site` and `museum-dashboard` use the same engine.

```
shared/
├── i18n/
│   ├── index.ts              ← Core functions
│   └── LanguageContext.tsx   ← React context + useLanguage() hook
├── locales/
│   ├── en.json               ← Source of truth
│   ├── ar-EG.json
│   ├── de.json
│   ├── es.json
│   ├── it.json
│   └── fr.json
├── components/
│   └── LanguageSwitcher.tsx  ← Globe button UI
└── utils/
    └── exhibitTranslations.ts ← Per-exhibit DB translation helper
```

## Core API (`shared/i18n/index.ts`)

### Types

```ts
type Locale = "en" | "ar-EG" | "de" | "es" | "it" | "fr";
type Dictionary = Record<string, any>;
```

### Functions

| Function                       | Description                                                          |
| ------------------------------ | -------------------------------------------------------------------- |
| `getDictionary(locale)`        | Returns the translation dictionary for a locale (falls back to `en`) |
| `createT(dict)`                | Returns a `t(key, vars?)` function bound to a dictionary             |
| `isRTL(locale)`                | Returns `true` for `ar-EG`                                           |
| `getLocaleDisplayName(locale)` | Returns the human-readable name, e.g. `"العربية"`                    |
| `getAvailableLocales()`        | Returns `["en", "ar-EG", "de", "es", "it", "fr"]`                    |
| `isValidLocale(str)`           | Type-guard to check if a string is a valid `Locale`                  |

### Translation key resolution

- Keys use dot notation: `"home.hero.title"`, `"nav.exhibits"`.
- `getNestedValue(dict, key)` traverses the JSON object.
- If the key is missing in the selected locale, it falls back to English.
- If missing in English too, a `console.warn` is emitted and the key string itself is returned.

### Interpolation

Use `{variable}` syntax in locale JSON:

```json
{ "footer.copyright": "© {year} Tourist Guidance Museum" }
```

```ts
t("footer.copyright", { year: "2025" });
// → "© 2025 Tourist Guidance Museum"
```

## React Context (`shared/i18n/LanguageContext.tsx`)

### Provider

Wrap the app root in `<LanguageProvider>`:

```tsx
// app/layout.tsx
import { LanguageProvider } from "../../shared/i18n/LanguageContext";

export default function RootLayout({ children }) {
  return (
    <html suppressHydrationWarning>
      <body>
        <LanguageProvider>{children}</LanguageProvider>
      </body>
    </html>
  );
}
```

### `useLanguage()` hook

```ts
const { t, locale, setLocale, isRTL } = useLanguage();
```

| Value       | Type                                                               | Description                                  |
| ----------- | ------------------------------------------------------------------ | -------------------------------------------- |
| `t`         | `(key: string, vars?: Record<string, string \| number>) => string` | Translate a key                              |
| `locale`    | `Locale`                                                           | Current active locale                        |
| `setLocale` | `(locale: Locale) => void`                                         | Switch language; persisted to `localStorage` |
| `isRTL`     | `boolean`                                                          | `true` when current locale is `ar-EG`        |

### Persistence & RTL

On `setLocale`:

1. Saves to `localStorage` key `"preferred-language"`.
2. Sets `document.documentElement.dir = "rtl" | "ltr"`.
3. Sets `document.documentElement.lang = locale`.

On initial render, the saved localStorage value is restored.

## Using Translations in Components

```tsx
"use client";
import { useLanguage } from "../../../shared/i18n/LanguageContext";

export default function MyComponent() {
  const { t, isRTL } = useLanguage();
  return <h1>{t("home.hero.title")}</h1>;
}
```

## Per-Exhibit Database Translations (`exhibitTranslations.ts`)

Exhibits store translations directly in the database as `jsonb` columns:

- `title_translations`: `{ "ar-EG": "...", "de": "...", ... }`
- `description_translations`: same shape

Use `getExhibitTranslation(exhibit, locale)` to resolve:

```ts
import { getExhibitTranslation } from "../../shared/utils/exhibitTranslations";

const { title, description } = getExhibitTranslation(exhibit, locale);
```

Fallback chain: selected locale → English → original `title`/`description` fields.

## Language Switcher Component

**File:** `shared/components/LanguageSwitcher.tsx`

A globe icon button that opens a dropdown listing all 6 languages.  
Calls `setLocale(locale)` on selection.  
Used in both `public-site` Navbar and `museum-dashboard` Navbar.

## Adding a New Language

1. Create `shared/locales/xx.json` — copy `en.json` and translate all values.
2. In `shared/i18n/index.ts`:
   - Import: `import xx from "../locales/xx.json";`
   - Add to `Locale` type: `| "xx"`
   - Add to `dictionaries`: `xx: xx`
   - Add to `getLocaleDisplayName`: `xx: "Language Name"`
   - Add to `getAvailableLocales()` return array
3. If RTL, add the locale to the `isRTL` check.
4. The `LanguageSwitcher` will automatically show the new locale.
